Descripción General del Proyecto

El proyecto busca desarrollar un sistema de análisis y diseño para la administración y gestión de ventas del supermercado Hipermaxi, ubicado en La Paz, Bolivia. Este sistema tiene como propósito mejorar la eficiencia en el manejo de inventarios y las ventas, optimizando las operaciones en sus sucursales.

Análisis Estructurado

El análisis estructurado del sistema permite descomponer las funciones clave del supermercado Hipermaxi para identificar los procesos principales y garantizar que cada uno sea implementado adecuadamente. Este enfoque utiliza dos modelos principales: el modelo ambiental y el modelo de comportamiento.

Modelo Esencial

Modelo Ambiental
  • Declaración de Propósitos (DP): Define los objetivos principales del sistema, como la optimización del control de inventarios, la gestión de ventas, y la relación con proveedores.
  • Lista de Acontecimientos (LA): Identifica los eventos clave del sistema, como ventas realizadas, actualizaciones de inventario, y órdenes de compra a proveedores.
  • Diagrama de Contexto (DC): Representa las interacciones principales entre el sistema y los actores externos, como clientes, administradores y proveedores.
Modelo de Comportamiento
  • Diagrama de Flujo de Datos (DFD): Muestra cómo la información fluye entre los diferentes procesos del sistema, como ventas, actualizaciones de inventarios y generación de reportes.
  • Diagrama Entidad/Relación (DER): Define las relaciones entre las entidades clave del sistema, como productos, clientes y proveedores.
  • Diagrama de Datos (DD): Detalla los datos utilizados y producidos por el sistema, como registros de ventas, catálogos de productos, y órdenes de compra.
  • Diagrama de Transacciones (DTS): Representa las transacciones principales del sistema, como la venta de productos, actualizaciones de inventarios y pedidos a proveedores.

Alcance y Objetivos

Este proyecto tiene como alcance implementar un sistema que abarque la gestión de ventas, el control de inventarios y la administración de proveedores, asegurando un flujo continuo de productos en las sucursales y mejorando la experiencia tanto de los clientes como de los administradores del supermercado Hipermaxi.

Información del Supermercado Hipermaxi

Hipermaxi, ubicado en Av. Arce NRO 2519 Condominio Torres del Poeta, La Paz, Bolivia, cuenta con 11 sucursales en la ciudad. Ofrece una amplia variedad de productos en categorías como alimentos, bebidas, productos de limpieza y artículos de hogar, atendiendo a más de 5,000 clientes registrados.

¿Qué es el Análisis Estructurado?

El análisis estructurado es una metodología que permite descomponer sistemas complejos en partes simples y organizadas, facilitando su diseño y documentación. Su enfoque principal es garantizar que los procesos y datos sean consistentes y estén alineados con los objetivos del sistema.

Modelo Ambiental

El modelo ambiental define el alcance del sistema y su interacción con los agentes externos. Incluye los objetivos clave, los eventos principales que activan procesos y las relaciones con otras entidades.

  • Declaración de Propósitos (DP): Este modelo tiene como objetivo garantizar una gestión eficiente del inventario, procesar ventas con rapidez y mantener un registro actualizado de clientes y proveedores.
  • Lista de Acontecimientos (LA): Los eventos incluyen:
    • Registro de nuevos productos en el inventario.
    • Actualización de cantidades en stock después de cada venta.
    • Generación de reportes mensuales de ventas y stock.
    • Notificación automática al administrador sobre niveles bajos de inventario.
  • Diagrama de Contexto (DC):

    Representa cómo el sistema interactúa con los agentes externos. Los principales actores son:

    • Clientes: Realizan compras y reciben comprobantes de pago.
    • Administradores: Gestionan el inventario y supervisan las ventas.
    • Proveedores: Suministran productos al sistema.
    • Sistema de Inventario: Registra y actualiza el stock en tiempo real.

Modelo de Comportamiento

El modelo de comportamiento describe los flujos internos del sistema, las relaciones entre las entidades principales y los datos que se procesan durante la operación del supermercado.

  • Diagrama de Flujo de Datos (DFD): Representa cómo la información fluye entre los procesos del sistema.
  • Diagrama Entidad/Relación (DER): Describe las conexiones entre las entidades principales: Productos, Clientes, Administradores y Proveedores.
  • Diagrama de Datos (DD): Detalla la estructura de los datos, como ID de productos, categorías y niveles de inventario.
  • Diagrama de Transacciones (DTS): Muestra los procesos clave, como agregar productos al inventario y registrar ventas.




Modelo Ambiental para nuestro sistema


a) DECLARACIÓN DE PROPÓSITOS:

El sistema tiene como objetivo principal gestionar las operaciones de venta y el control de inventarios en el supermercado Hipermaxi. Esto incluye la administración de productos, precios, existencias, compras de clientes, proveedores, ofertas y reportes de ventas. Además, permite mejorar la experiencia del cliente, optimizar los procesos internos y asegurar la disponibilidad de productos.

b) LISTA DE ACONTECIMIENTOS:

Eventos relacionados con las ventas:

  1. Un cliente selecciona productos para comprar.
  2. Un cliente realiza el pago en caja.
  3. Se genera un comprobante de venta.
  4. Se realiza una devolución de producto por parte del cliente.

Eventos relacionados con la gestión de inventarios:

  1. El inventario de un producto disminuye tras la venta.
  2. Un empleado actualiza las existencias de un producto.
  3. Se realiza el ingreso de un nuevo producto al sistema.
  4. Se detecta un producto con stock mínimo y se notifica al proveedor.

Eventos relacionados con proveedores:

  1. Se realiza un pedido de reposición de productos a un proveedor.
  2. Se registra la recepción de un pedido de un proveedor.

Eventos relacionados con ofertas y promociones:

  1. Un administrador define una oferta o descuento para un producto.
  2. Se actualizan los precios de productos en oferta.

Modelo de Comportamiento

  • Diagrama de Flujo de Datos (DFD):

    El sistema coordina módulos de inventario, ventas y pagos para gestionar productos, transacciones y reportes en Hipermaxi. Conecta proveedores, clientes y administración para mantener el flujo de mercancías, ventas y conciliaciones financieras.

  • Diagrama Entidad/Relación (DER):

    El diagrama entidad-relación describe las interacciones entre usuarios, ventas, productos e inventarios en el sistema de gestión Hipermaxi, destacando cómo los clientes realizan compras, los administradores consultan reportes, y los proveedores suministran productos al inventario para mantener el flujo operativo.

  • Diagrama de Datos (DD):

    El diagrama de datos define las relaciones entre entidades clave como clientes, productos, ventas, pagos, promociones e inventarios en el sistema. Cada entidad contiene atributos específicos, y sus relaciones garantizan la trazabilidad de operaciones como pedidos, devoluciones y generación de reportes.

  • Diagrama de Transacciones (DTS):

    El diagrama de transacciones muestra cómo interactúan los módulos del sistema, destacando la gestión de inventarios, promociones, pagos y devoluciones, vinculados a clientes, cajeros, proveedores y administradores. Cada flujo representa procesos como actualización de stock, registro de compras y generación de reportes, asegurando un manejo integral de operaciones en Hipermaxi.

Modelado Orientado a Objetos

El modelado orientado a objetos del sistema del supermercado Hipermaxi refleja la interacción entre las principales entidades del negocio, sus relaciones y comportamientos. A continuación, se describen las principales clases y sus asociaciones:

Clase Reporte

La clase Reporte está asociada con:

  • Inventario: Relación de asociación para generar reportes de existencias y movimientos de productos.
  • Producto: Relación de asociación para detallar información de los productos en reportes específicos.
  • Empleado: Relación de asociación para incluir detalles del personal responsable en ciertos reportes.

Clase Inventario

La clase Inventario utiliza una relación de composición con:

  • Producto: Cada producto está directamente contenido en el inventario, estableciendo una dependencia fuerte.

Clase Venta

La clase Venta incluye las siguientes relaciones:

  • Producto: Relación de agregación para incluir múltiples productos en una transacción.
  • Empleado: Relación de asociación para registrar al empleado que procesó la venta.
  • Cliente: Relación de asociación para identificar al cliente involucrado en la transacción.

Clase Producto

La clase Producto tiene las siguientes relaciones:

  • CarritoDeCompra: Relación de asociación para incluir productos seleccionados por el cliente.
  • CategoríaProducto: Relación de asociación para clasificar productos según su categoría.

Clase Persona

La clase Persona es una clase general que hereda a:

  • Proveedor: Relación de herencia para representar a los proveedores del supermercado.
  • Empleado: Relación de herencia para gestionar a los empleados del sistema.
  • Cliente: Relación de herencia para manejar a los clientes registrados.

Este modelado orientado a objetos permite establecer una estructura clara y modular del sistema, facilitando su desarrollo y mantenimiento.
